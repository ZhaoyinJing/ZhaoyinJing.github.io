哈哈，今天上课听老师发言有感

  • 不要忙，要勤

    • 忙从字形来看，代表心亡。为何会心亡？因为忙碌使得我们没有时间思考，没有时间静下心来阅读。

    • 忙是被动的，勤是主动的，勤代表着一种主观能动性，正所谓天道酬勤。

  • 不要总拘泥于专业知识

    • 理工科的学生提升自己的专业能力这是很有必要的，但不能一昧地追逐本专业
    • 可能你只需要花费60%的精力就可以掌握一门专业课程的80%，但想要掌握透彻，需要更进一步将时间花费在这上面。但你如果将这些时间用来干些别的，比如思考、阅读，或者其他感兴趣的等等，你可能更加博学，生活更加有意义。

  • 学会创造财富

    人生有三大自由：健康自由，财务自由，时间自由

    • 尽量有两种不同来源的收入
      • 如果只有专业收入，那么人生会很平庸
    • 追求物质财富的同时也别忘了追求精神财富

hexo显示本地图片

发表于 2020-11-08 分类于 工具使用 ， Hexo

今天真是高产(我又遇到了问题)，哈哈，因为我将上一篇博文deploy之后发现图片加载失败…真的很吐血。去网上一搜索，嗯，好像不少人都遇到了这个问题，解决方法看似一大堆实则无效或过时不匹配，绝大多数资料都是安装hexo-asset-image的插件(ps. 很多资料提供的插件下载地址都不太对)，亲测对于markdown图片路径没有用啊！比出现问题更折磨人的是遇上一堆错误的解决方法…

我想要寻找在本地和网页上都能显示的办法，终于发现了一款插件hexo-image-link，是将markdown图片路径转换为asset_img语法，使得图片能够同时显示在typora和hexo上。

具体步骤

  • 修改_config.yml中的post_asset_folder: true
  • $ npm install hexo-image-link --save
  • 如果 npm下载比较慢的话，尝试 cnpm下载
    • $ npm install -g cnpm --registry=https://registry.npm.taobao.org
    • $ cnpm install hexo-image-link --save
  • 修改md文件中的图片路径

大功告成！

赶紧generate and deploy验证一下，完美！

搭建好自己的博客，总想对其美化一番，一路捯饬下来，也踩了不少坑。于是用博文记录下如何正确配置美化nexT主题，希望能够帮助到有缘看到此博的小白哈哈（好吧我也是个小白）…

从nexT-7.3.0开始，采用数据文件将配置与主题分离，便于后续nexT版本更新。

更改图片背景

发现目前网上流行的解决方案还是在hexo/themes/next/source/css/_custom/_custom.styl中进行配置。

新版本需要新建hexo/source/_data/styles.styl，即在hexo目录的source文件夹下新建_data文件夹，文件夹中新建styles.styl。

注意：是新建在hexo目录下source文件夹里，而不是主题文件next目录下的source文件夹。

打开styles.styl，添加如下：

  • background-image:url 为图片路径，可以直接使用链接，也可以是图片路径（将自定义图片放入hexo\public\images路径下）

  • background-repeat：若果背景图片不能全屏，那么是否平铺显示，充满屏幕

  • background-attachment：背景是否随着网页上下滚动而滚动，fixed 为固定

  • background-size：图片展示大小，这里设置 100%，100% 的意义为：如果背景图片不能全屏，那么是否通过拉伸的方式将背景强制拉伸至全屏显示

“所谓博客，不过是孤芳自赏”

在特殊的日子20200202这一天搭建好属于自己的博客，写下第一篇博文纪念哈。

我想，每个人写博客的初衷是希望自己的文章能够帮助他人解决问题，但更多时候更像是对自己所学所想进行记录，在浩瀚的互联网中留下自己的足迹，自赏自悦。

王阳明有言：“你未看此花时，此花与汝心同归于寂，你来看此花时，则此花颜色一时明白起来。便知此花不在你的心外”。博客亦如此，因你看见，所以存在。

生活中总有很多时候感慨颇多，想提笔抒发胸臆，但往往以费时费力为逃避借口，便一再搁置。过段时日却再也找不回当初所想所感。

此篇结束。未来的日子，待续…
